Linux根目录空间不足解决

2018-04-23   Linux 

解决Linux根目录空间不足问题

环境：windows10 1803 + ubuntu 16.04双系统

当初装双系统是还不了解linux的包安装机制，/目录就只分了10g，于是当我愉快地搭完DL环境后，用了一段时间，突然被win丑到，决定再愉快地装了一套wine软件以及网易云等日常使用工具后，我发现我的根目录已经只剩200M了，已经寸步难行

网络上给出修改linux分区的方法是使用GParted，不用格式化可以直接对linux分区进行修改，随着软件更新，在我真正使用过程中发现某些教程还是有些坑，于是决定自己写一篇新的。

1.制作GParted启动盘

1.1 烧写启动盘的软件：

  • ubuntu: tuxboot 
sudo apt-add-repository ppa:thomas.tsai/ubuntu-tuxboot
sudo apt-get update
sudo apt-get install tuxboot
  • windows:UltraOS

1.2 下载GParted

官方下载页

如果64位版U盘启动后出现“gparted启动错误”等提示，重新下载32位版进行烧写，不要再去褥BIOS了（我在这里被坑了很久=。=）

1.3 开始烧写

  • 使用tuxboot：

    pre Downloaded选择镜像文件→开始刻录

  • 使用UltraISO:

    打开镜像文件→启动→写入硬盘镜像

2.使用GParted调整分区

2.1 设置U盘启动

不同机型不同操作，自行百度

2.2 启动GParted

  • 选择GParted Live
  • 按键映射有关的东西，不管，直接回车
  • 语言选择，凭个人爱好
  • 选择模式，默认为图形界面，按回车即可
  • 进去之后点击GParted图标，开始修改分区

从邻近分区先移出空余部分，再与\分区进行合并，需要注意的是GParted只能合并磁盘上连续的分区，如果想把有间隔的分区调整，需要分步搬过去，调整完后点击“Apply”，完成后就可以愉快地继续改造你的linux了
